# HTML UI Spec

## 目的

- 公開しても差し支えない、要約中心の旅行ページにする
- 人が一目で見やすいことを優先する

## 内容

- 載せるのは `概要` `エリア` `カテゴリ` `持ち物/準備` `予算の大枠` まで
- `時刻入り日程` は載せない
- `店や施設の詳細ページ` は作らない
- `ホテル / 航空券 / 予約 / 連絡先 / 実名` は載せない
- 実調査メモや内部判断理由は載せない

## 構成

- 基本は `1ページ完結`
- 先頭に `Hero`
- その下に `3〜6個の要約カード`
- セクション数は `多くても4つ前後`

## 見た目

- 明るい配色を基本にする
- 情報はカードで区切る
- 角丸は大きめ、影は弱め
- モバイルで先に見やすいことを優先する
- 文字サイズ差で強弱をつけ、装飾は増やしすぎない

## 振る舞い

- 操作は最小限にする
- タブや切り替えは必要な場合だけ使う
- `localStorage` を使う場合も、個人情報は保存しない
